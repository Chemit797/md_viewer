# 第二部分：方法选择

## [3] 怎么融入机理建模 —— 从机理建模总览到 GEM

### 3.1 融合范式：为什么不能纯数据驱动

**[A] 纯数据驱动（黑盒 AI）**
```
输入特征 → 大量数据训练 → 模型预测
优点：实现简单，性能可能很高
缺点：数据根本不够
      治疗性多肽-ADA 配对数据公开 < 30 对
      IEDB 中治疗性多肽 T 细胞实验数据 = 0
      → 纯数据驱动在这个问题上根本走不通
```

**[B] 纯机理模型（白盒）**
```
用 ODE / GEM / ABM 等数学工具刻画生物过程
优点：可解释，机理清楚
缺点：参数难拟合，预测精度受限于机理理解程度
      → 单独用机理模型预测能力有限
```

**[C] 机理 + AI 融合（我们的路线）**
```
用机理模型提供"归纳偏置"：
    把已知的生物学规律（代谢-命运因果链）
    直接编码进模型结构
    → 减少对数据量的依赖

用 AI 学习剩余的映射：
    机理模型输出的特征 → 简单分类器
    50-100 个样本足够训练逻辑回归
```

落到项目场景，"纯数据驱动"的代表是 EpiVax ISPRI（用大量 mAb-ADA 数据训练黑盒模型）。它的局限是：方法和数据都不公开，性能宣称（如"6 倍相关性提升"）无法独立验证，且对治疗性多肽没有任何验证数据。

---

### 3.2 现有机理建模方法全景

| 方法 | 适合建模什么 | 在本项目的角色 |
|---|---|---|
| ODE（常微分方程） | 信号通路动力学（TCR→mTOR→HIF-1α） | 辅助：Step 2的桥梁层（Martínez-Méndez 2021） |
| GEM + FBA | 细胞内代谢网络稳态通量 | **核心**：Step 3的主工具 |
| ABM（智能体模型） | 细胞群体互作（T细胞-B细胞-APC） | 不做：工程量过大 |
| scRNA-seq 嵌入 | 细胞整体状态表征 | 仅数据预处理辅助，不进主pipeline |
| 动力学FBA（dFBA） | 代谢通量的时间动态 | 不做：第一版用稳态FBA，够用就不加复杂度 |

---

### 3.3 GEM选择的逻辑链

```
项目主战场 = T 细胞活化后（6-72h）的代谢重编程
    ↓
"代谢重编程"是真正的代谢过程
    （糖酵解通量、FAO 通量、谷氨酰胺通量都可量化）
    ↓
代谢通量在数学上最自然的建模工具就是 GEM
    （ODE 适合信号通路，GEM 适合代谢网络）
    ↓
T 细胞场景有现成可用的 GEM：CD4T1670 + Sen 2021
    ↓
代谢-命运因果链有实验证据：
    mTORC1 → 糖酵解 → Tfh 倾向
    AMPK → FAO → Treg 倾向
    ↓
GEM 是当前最自然的起点
```

### 3.4 T 细胞 GEM 的现状

**GEM 家谱：**

```
Human-GEM（通用人代谢模板）
    ├─ ~13,000 反应，覆盖全身代谢通路
    └─ 项目的"基础模板"
        ↓
        加上 CD4+ T 细胞转录组约束
        ↓

CD4T1670（Han 2016, PMID 26646474）
    ├─ 1670 个代谢反应（naive CD4+ T 细胞）
    ├─ 覆盖：糖代谢、TCA、脂肪酸代谢、谷氨酰胺代谢
    └─ 用途：提供静息态代谢基线
        ↓
        加上各亚型分化条件的转录组约束
        ↓

Sen 2021 亚型 GEM（Cell Reports, PMID 34758307）
    ├─ Th1 / Th2 / Th17 / iTreg 四个亚型（已极化稳态）
    ├─ 整合了脂质组学验证（鞘糖脂通路）
    └─ 没有 Tfh，没有激活早期状态 ← 项目要补的方向
        ↓

激活期 CD4+ T 细胞 GEM ← 项目原创贡献
    ├─ 描述 TCR 激活后 6-72h 的代谢状态
    ├─ 这个时间窗口是分化命运决策期
    ├─ 文献中无此时间窗口的 GEM（PubMed + bioRxiv 检索确认）
    └─ 构建方法：GSE212270 / Soskic 2022 + tINIT + Human-GEM
```

**注意**：我们建的是**激活期 CD4+ T 细胞 GEM**，不是 Tfh GEM，不是 Treg GEM。分化方向是通过在这个 GEM 上施加不同的 mTORC1/AMPK 约束来预测的，不是通过构建两个不同的 GEM 来比较。

### 3.5 GEM 在项目里的具体使用方式（数据流详解）

**完整数据流：**

```
[阶段 1] 构建激活期 CD4+ T 细胞 GEM

    数据来源（按优先级）：
        优先：GSE212270（GEO 公开，0/12/24h scRNA-seq，CD4+ 专一）
        最优：Soskic 2022（EGA 申请，0/16/40h/5d，655,349 个细胞）

    构建步骤：
        提取 12-24h 激活期 CD4+ T 细胞的基因表达谱
            ↓
        tINIT 算法 + Human-GEM 裁剪
            （基因表达高 → 对应反应保留）
            （基因表达低 → 对应反应删除）
            ↓
        激活期 CD4+ T 细胞代谢子网络（约 1000-2000 个反应）

    验证：
        与 CD4T1670（naive 态）对比，激活期 GEM 应当：
            糖酵解相关基因活性更高
            线粒体生物发生相关基因更活跃
        与已知激活期 T 细胞代谢文献比对（Jones 2017, PMID 29170670）

---

[阶段 2] 从多肽序列到激活信号

    输入：Step 1 输出的 IC50 值

    转换1：IC50 → 激活强度参数（0-1）
        方法：自设计归一化函数（sigmoid 等）
        ❓ 不确定点：函数形状需消融实验确定

    转换2：激活强度 → mTORC1/AMPK 活性
        工具：Martínez-Méndez 2021 ODE 模型（51节点）
        输出：mTORC1 活性（0-1）+ AMPK 活性（0-1）
        ❓ 不确定点：ODE 输入格式为 avidity，
                     与 IC50 之间的转换仍是自设计

    转换3：mTORC1/AMPK 活性 → GEM 约束上下界调整
        mTORC1 高 → 糖酵解相关反应通量上界调高
        AMPK 高   → FAO 相关反应通量上界调高
        ❓ 不确定点：调高比例需消融实验确定

---

[阶段 3] FBA 求解代谢通量

    在同一个激活期 GEM 上，施加阶段 2 得到的约束，跑 FBA：

    工具：COBRApy（Python）

    场景 A（强激活信号，IC50 小）：
        高 mTORC1 约束
            ↓ FBA 求解
        糖酵解通量偏高，FAO 通量偏低
        → 代谢偏向 Tfh 分化

    场景 B（弱激活信号，IC50 大）：
        高 AMPK 约束
            ↓ FBA 求解
        FAO 通量偏高，糖酵解通量偏低
        → 代谢偏向 Treg 分化

    输出：代谢通量向量
        关键通量指标：
            糖酵解总通量（HK2、PFK、PKM 等反应的通量）
            FAO 总通量（CPT1A、HADHA 等反应的通量）
            谷氨酰胺氧化通量
            （具体选哪些指标，根据数据分析确定）

---

[阶段 4] 接入分类器

    输入特征：
        来自 Step 1：IC50 值 + Tregitope 标记
        来自阶段 3：代谢通量向量（关键通量指标）

    模型：逻辑回归
        原因：训练数据 50-100 样本，简单模型为宜
              GEM 提供了强机理归纳偏置，分类器本身不需要复杂

    训练标签：复旦 PBMC 体外实验
        T 细胞增殖指数 SI + Tfh 标志物（CXCR5+ PD-1high BCL-6+）

    输出：Tfh 倾向分数（0-1）= ADA 风险代理
```

### 3.6 三个关键设计决策

**决策 A：为什么只建一个 GEM，而不是建 Tfh 和 Treg 两个 GEM 来比较**

```
错误做法：
    用已分化 Tfh 细胞的数据建 Tfh GEM
    用已分化 Treg 细胞的数据建 Treg GEM
    然后比较同一条多肽在两个 GEM 里的通量
    → 这是循环论证：用分化结果来预测分化方向

正确做法：
    只建一个 GEM：激活期 CD4+ T 细胞（6-72h，分化命运尚未固定）
    在这个 GEM 上，通过改变 mTORC1/AMPK 约束来模拟不同激活强度
    → 代谢通量的偏向就是分化倾向的预测
    → 没有循环，因果方向正确
```

**决策 B：静态 GEM（稳态 FBA）还是动态 GEM（时间序列）**

```
第一版：静态 GEM（稳态 FBA）
    优点：方法成熟，CD4T1670 / Sen 2021 都是稳态，有先例
    代价：不直接刻画 6-72h 内的时间动态

如何近似：
    用激活期（12-24h）的转录组数据构建 GEM
    代表"分化决策时间窗口"的代谢状态
    通过 mTORC1/AMPK 约束的强弱模拟不同激活信号

第二版可选扩展（不在当前计划内）：
    引入 dFBA（动态通量平衡分析）
    刻画 0 → 6 → 24 → 72h 的代谢变化轨迹
```

**决策 C：激活期 GEM 的数据选择**

```
优先：GSE212270（立即可用）
    CD4+ naive + memory，0/12/24h，10x scRNA-seq
    GEO 公开，本周可下载

最优：Soskic 2022（需申请）
    CD4+ 多亚群，0/16/40h/5d，655,349 个细胞，119 名供体
    EGA 申请，2-4 周审批，本周提交

❓ 待核查（下载后第一件事）：
    检查 CPT1A、HADHA 等 FAO 相关基因在 12-24h 时间点的表达量
    如果 10x 3' 文库 dropout 导致这些基因覆盖不足
    → GEM 里的 FAO 通路会被错误地关掉
    → 需要用 bulk RNA-seq 补充（GSE197067 作辅助参考）
```

### 3.7 GEM 的局限（需在论文中诚实声明）

**[局限 1] 稳态假设**
```
GEM 求解的是稳态通量，不直接刻画时间动态
实际 T 细胞在 6-72h 内代谢是动态变化的
补救：用激活期 12-24h 的快照作为近似
      第二版可引入 dFBA
```

**[局限 2] 信号通路不在 GEM 范围内**
```
TCR 信号、mTOR 通路、HIF-1α 调控等
都不是代谢反应，不在 GEM 反应清单里
补救：把信号通路活性作为 GEM 输入的约束条件
      即 mTORC1/AMPK 活性 → 调整通量上下界
      这部分通过 Martínez-Méndez ODE 模型桥接
```

**[局限 3] Step 2 全程自设计，误差累积**
```
IC50 → 激活强度 → mTORC1/AMPK → GEM 约束
这条链上三个转换都是自设计，没有直接文献先例
每一步都会引入误差，误差向下传递
补救：消融实验验证每一步的转换方式
     论文中诚实报告 Step 2 是项目技术不确定性最高的部分
```

**[局限 4] Tfh-Treg 激活期代谢可区分性待确认**
```
❓ 这是整个 GEM 模块的核心待确认问题

已知：成熟稳态下 GC-Tfh 和 Treg 代谢主轴相似
      （Mayberry 2022 综述）
未知：激活期（6-72h）两者代谢轨迹是否可区分

如果可区分 → GEM 通量有区分力，整个 Step 3 有价值
如果不可区分 → Step 3 需要重新设计或移除
              退化到 Step 1 + Step 4 的简化版本

这个问题必须在开会时问 Alan
```

### [决策点] 为什么选 GEM 而不是 ODE / scRNA-seq embedding

```
┌─ [决策点] GEM 是机理建模的最佳起点 ─────────────────────┐
│                                                          │
│ 结论：项目第一版用 GEM 作为机理建模核心，               │
│      ODE 作为 Step 2 的信号通路桥梁（辅助）             │
│      scRNA-seq embedding 不进主 pipeline                 │
│                                                          │
│ 关键证据：                                               │
│   [证据 1] Sen et al. 2021（PMID 34758307）              │
│            证明 GEM 能定量区分 Th1/Th2/Th17/iTreg 代谢   │
│            并经过实验验证（SPT 敲降 → IL-17A 下降）      │
│                                                          │
│   [证据 2] Leddon & Sant 2012（PMID 23077537）           │
│            pMHC-II 稳定性（与 IC50 相关）                │
│            与 Tfh 生成强度正相关                         │
│            支持"IC50 → 激活信号 → 代谢偏向"这条链        │
│                                                          │
│   [证据 3] scGPT 等单细胞基础模型的系统评估              │
│            在扰动预测上未显著超过简单基线                │
│            说明它们不能替代机理建模                      │
│            → scRNA-seq embedding 不进主 pipeline         │
│                                                          │
│ 逻辑链：                                                 │
│   项目主战场是代谢重编程                                 │
│       → GEM 是代谢网络的最佳建模工具                     │
│       → CD4T1670 + Sen 2021 已有方法论模板               │
│       → 激活期 GEM 是文献空白，是项目原创贡献            │
│       → ODE 桥接信号通路（Martínez-Méndez 2021）         │
│       → 简单分类器在小数据下优于复杂模型                 │
│       → GEM + 逻辑回归是当前最合理的架构                 │
│                                                          │
│ 各方法的最终角色：                                       │
│   GEM     → 核心：提供代谢通量特征（Step 3）             │
│   ODE     → 辅助：Step 2 的信号通路桥梁                  │
│   scRNA-seq embedding → 数据预处理辅助，不进主 pipeline  │
│   逻辑回归 → 分类器（Step 4）                            │
│   ABM / 多尺度 → 不实做，工程量过大                     │
│                                                          │
│ 证据强度：高（框架方向），中（Step 2 定量细节）          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```
