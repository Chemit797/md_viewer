# 活化态 CD4⁺ T (Tfh) 细胞基因组尺度代谢模型 (GEM) 项目进展报告

**报告时间**：2026年6月
**项目目标**：构建一张“分化前活化态 CD4⁺ T 细胞”的细胞特异性代谢网络图（GEM），作为底层图结构拓扑特征，对接下游图神经网络 (CytoGem GNN) 预测 Tfh/Treg 免疫原性分化命运。

---

## 一、 基于参考文献确定的 GEM 构建思路

通过对多篇核心系统生物学文献的拆解与重构，我们确立了**“自上而下的任务驱动裁剪”**作为本次建图的核心范式。具体的建图思路综合了以下三篇关键文献的方法论：

### 1. 核心裁剪范式：任务驱动的表达量映射 (基于 Agren et al. / tINIT 算法)
* **思路描述**：我们放弃了仅凭基因表达有无进行“一刀切”的生硬方法。采用 tINIT（以及现在的 ftINIT）算法，该算法允许我们将通用参考模型（Human-GEM）作为底图。我们将 Tfh 细胞的单细胞转录组（转化后的 Pseudobulk 数据）转换为表达正负评分（Score）。
* **技术关键**：算法在尽力删减不表达的反应时，**受到“必须完成设定代谢任务”的强约束**。如果删除某条不表达反应会导致细胞无法生长或完成关键代谢，算法会触发 Gap-filling（缝补）机制强行保留该通路，从而保证生成的网络在结构和生物学意义上的连通性。

### 2. 活化态生理特征的锚定：自定义特征任务 (基于 Sen 2021)
* **思路描述**：单纯的转录组数据无法完全反映 T 细胞活化时的代谢重编程（如转录后调控）。Sen 等人在构建活化态 Th0 模型时，利用基础代谢任务做强约束。
* **我们如何借鉴**：我们将定义专属的 **Tfh_tasks** 任务表。除了要求模型满足巨噬细胞级别的 Biomass 生成外，强制要求模型必须能在有氧状态下摄取葡萄糖并**分泌 L-乳酸 (L-lactate)**，以此在网络拓扑结构中强行保留**有氧糖酵解 (Warburg effect)** 的骨架。同时开放脂肪酸吸收约束，保留 FAO 氧化潜能。

### 3. CD4⁺ 亚型特异性裁剪细则：同工酶与连通性 (基于 Puniya 2021)
* **思路描述**：Puniya 等人提供了极为详尽的 CD4+ 亚型 GEM 构建规则参数。
* **我们如何借鉴**：
  * **同工酶容忍原则 (Isozyme/GPRs)**：在反应拥有多个同工酶时，只要 GSE212270 数据中检测到一个同工酶表达，即保留整个反应及其完整的基因-蛋白质-反应 (GPR) 映射。
  * **阈值共识**：针对单细胞测序深度问题，采用群体覆盖度（如在 >50% 的 Tfh 细胞中表达）来设定稳健的二值化阈值。
  * **死胡同清理**：构建完毕后，严格执行 `removeDeadEnds` 操作，移除在生物体内无源无汇的孤立反应碎片，确保交给下游 GNN 的每一条边都存在物理意义。

## 二、 目前已完成的工作

截至目前，我们已经跨越了“纸上谈兵”阶段，完成了底层逻辑对齐和工程脚手架的搭建，具备了直接进行计算的条件。具体已完成：

1. **确定了技术路径与验证策略的“松绑”**
   * 明确了本项目的交付物是“图结构”而非“动态仿真器”。因此我们决定**取消为了让 FBA 通量预测完全闭合而进行的耗时且主观的过度手动 Gap-filling**。
   * 制定了以“关键拓扑通路是否存在”（文献对照检查）为主的第一层验证策略，极大加速了工程进度。

2. **制定了五步施工蓝图**
   * 输出了详细的工程指南文件 `tfh_gem_plan.md`，明确了：预处理 -> 定义任务 -> ftINIT 裁剪 -> 清洗优化 -> 特征验证的 SOP 流水线。

3. **开发了自动化全链路脚手架（双驱脚本）**
   * **`1_pseudobulk_GSE212270.R`（翻译器）**：编写了从单细胞 Seurat 对象到 MATLAB `data_struct` 的一键转换代码，内置了表达量二值化与 ENSEMBL ID 规范化功能。
   * **`2_build_tfh_gem.m`（代工厂）**：编写了基于 MATLAB/RAVEN 工具箱的 `ftINIT` 核心调度脚本，包含了任务挂载、环境边界开放、死端清理及网络输出为 `.mat`/`SBML` 的完整逻辑代码。

## 三、 接下来需要做什么 (Next Steps)

工厂已经组装完毕，接下来就是进料与开工。我们将按照以下步骤迅速推进：

### 1. 数据校验阶段：“第 0 道关卡”排雷（当务之急）
* **动作**：下载 GSE212270 数据集并提取 Tfh 亚群。
* **目的**：利用 Python/R 探查关键代谢基因：FAO 双雄（**CPT1A、HADHA**）与糖酵解双雄（**SLC2A1/GLUT1、HK2**）在该数据集中的非零覆盖度（Dropout 率）。
* **策略**：如果测序深度太浅导致覆盖度极低，为了避免 `ftINIT` 将其错误剪切，我们将在 `data_struct.mat` 生成步骤中引入“先验补丁”，强制将这些验证级特征基因的表达态锁定位 `1`。

### 2. 物料准备阶段：通用底图与特征任务装配
* **动作**：克隆下载完整的 `Human-GEM` (v1.12+) 最新版本仓库，提取其提供的官方参考模型 `Human-GEM.mat`。
* **动作**：提取官方 `metabolicTasks_Essential.txt`，在文件末尾手动追加 T 细胞活化特有任务（L-乳酸分泌通道、Biomass 前体组装通道），完成定制化测试。

### 3. 流水线执行与网络导出阶段
* **动作**：按顺序运行 `1_pseudobulk_GSE212270.R` 和 `2_build_tfh_gem.m`。
* **排错**：监控 `ftINIT` 算法的内存和运行状态，必要时回退至单纯的结构验证（不强制通量闭合）。
* **输出**：拿到包含关联矩阵（Stoichiometric Matrix S）的 `Tfh_GEM_final` 文件，检查格式是否满足下游 CytoGem 中超图构建的要求。

### 4. 模型质量审查阶段 (第一层验证)
* **动作**：在生成的网络拓扑中，查询是否成功保留了脂肪酸氧化通路节点和有氧糖酵解通路节点。如果确认存在，则模型初步构建宣告成功，直接移交 GNN 进入表征学习阶段。
