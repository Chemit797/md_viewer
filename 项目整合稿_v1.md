## 1. 做哪种细胞？

### 1.1 为什么是 T 细胞而不是其他免疫细胞

要回答这个问题，必须先理解 **ADA（抗药抗体）形成的完整链路**：

```
ADA 形成的完整链路：

[1] 多肽进入体内
        ↓
[2] APC（抗原呈递细胞，如树突细胞 DC、巨噬细胞）吞噬多肽
        ↓
[3] APC 内部用 Cathepsin 酶切割多肽为 13-25 aa 片段
        ↓
[4] APC 把片段挂载到自己表面的 MHC-II 分子上展示
        ↓
[5] CD4+ T 细胞用 TCR 识别 APC 展示的肽-MHC-II 复合物
        ↓
[6] T 细胞被激活，开始增殖 + 分泌细胞因子
        ↓
[7] T 细胞分化成不同角色（关键决策点！）
    ┌─────────────────┬─────────────────┐
    ↓                 ↓                 ↓
   Tfh             Treg             其他（Th1/Th2/Th17）
   ↓                 ↓
[8a] 跑到淋巴结    [8b] 抑制免疫应答
   滤泡区辅助
   B 细胞激活
   ↓                 ↓
[9a] B 细胞分化   [9b] 不产生 ADA
   为浆细胞
   分泌 ADA
```

```
在这条链上选择"做哪种细胞"的论证：

候选 1: B 细胞（ADA 的直接来源）
    → 不选。B 细胞是终末执行者，
       决定它行为的上游因素是 CD4+ T 细胞
       而且 B 细胞的代谢-命运因果证据弱于 T 细胞

候选 2: DC / 巨噬细胞（APC）
    → 不选。APC 的"决策"是相对机械的
       （摄取→加工→呈递）
       代谢状态对最终 ADA 影响小

候选 3: NK 细胞
    → 不选。与 ADA 形成无直接关系

候选 4: CD8+ T 细胞
    → 不选。负责细胞毒性应答，不参与 ADA

候选 5: CD4+ T 细胞 ←【选这个】
    → ADA 因果链的"瓶颈节点"
       没有 CD4+ T 帮助，B 细胞难以产生高亲和力 IgG（这是 Td-ADA 定义）
       代谢-命运因果证据最强
       已有 GEM 工具（CD4T1670, Sen 2021）
```

【判断】Alan 明确指令是"构造一个虚拟的 T 细胞"。这个指令对应的精确科学含义就是 **CD4+ T 细胞**，特别是在 APC-T 互动中决定 Tfh 还是 Treg 命运的过程。

### 1.2 为什么聚焦 Tfh vs Treg 这对决策

在 CD4+ T 细胞活化后，会分化成多种亚型。各亚型的功能与对 ADA 的影响：

```
CD4+ T 细胞分化方向：

Th1     ──→ 对抗细胞内病原（病毒等）           对 ADA：中等
Th2     ──→ 对抗寄生虫、过敏反应               对 ADA：中等
Th17    ──→ 对抗真菌、炎症反应                 对 ADA：间接
Treg    ──→ 抑制免疫反应，维持耐受             对 ADA：↓（降低）
Tfh     ──→ 在淋巴结滤泡区辅助 B 细胞产抗体    对 ADA：↑（直接驱动）

→ Tfh 和 Treg 是一对"冤家"：
   Tfh ↑ 则 ADA ↑
   Treg ↑ 则 ADA ↓
   
→ 项目的核心科学命题就是：
   "什么因素决定一个多肽更倾向激发 Tfh 还是 Treg？"
```

### 1.3 关于 Tfh 的关键术语

```
Tfh = T follicular helper cell（滤泡辅助 T 细胞）
    ├─ 名字来源：在淋巴结的"滤泡"（follicle）里帮 B 细胞
    ├─ 主转录因子：Bcl-6（决定 Tfh 命运的基因开关）
    └─ 标志蛋白：CXCR5（让它能找到滤泡）、PD-1（高表达）

GC-Tfh = Germinal Center Tfh（生发中心 Tfh）
    └─ 在淋巴结的"生发中心"里成熟的 Tfh，
       是辅助 B 细胞产抗体的最终形态
       
cTfh = circulating Tfh（外周循环 Tfh）
    └─ 血液中的 Tfh，未到达淋巴结
       代谢状态与 GC-Tfh 不同
```

## 2. 做这种细胞的什么过程？GEM 能贡献什么？

### 2.1 ADA 链路上 GEM 能力对照

这是项目最关键的判断之一：**GEM 不是万能的，必须找到它能真正贡献的环节**。

```
ADA 链路各环节 vs GEM 能不能建模：

环节                时间尺度    GEM 适合度    备注
─────────────────────────────────────────────────────
[1] 多肽进入        即时        ❌ 不适合     不是代谢问题
[2] APC 吞噬        分钟        ❌ 不适合     膜运输 / 内吞
[3] Cathepsin 切割  分钟-小时   ⚠️ 边缘       GEM 含蛋白水解但粒度粗
[4] MHC-II 呈递     小时        ❌ 不适合     蛋白质装配，非代谢
[5] T 细胞识别      分钟-小时   ❌ 不适合     信号通路（用 ODE）
[6] T 细胞活化      6-24 小时   ✅✅ 主战场   代谢重编程（糖酵解↑）
[7] T 细胞分化      24-72 小时  ✅✅✅ 核心   代谢-命运因果链
[8] Tfh 帮 B 细胞   数天        ❌ 不适合     跨细胞互作
[9] B 细胞产 ADA    数周        ❌ 不适合     不在 GEM 范畴
```

**核心识别**：在 ADA 形成全链条中，**GEM 真正能贡献的只有 [6]-[7] 这一段**。

### 2.2 项目主战场：T 细胞活化 + 分化

```
我们的项目只盯着这一段：
                              ↓
                  [6] T 细胞活化  [7] T 细胞分化

其他环节怎么办？
- [1]-[4]: 用现有工具处理
    ├─ NetMHCIIpan-4.3 算"哪些片段会被 MHC-II 呈递"
    ├─ Barra 2020 (NNAlign_MAC) 可作 MAPPs 训练参考
    └─ TLimmuno2 算"T 细胞激活概率"作为补充特征
- [5]: 用 TCR 结合预测工具（非项目核心）
- [8]-[9]: 不建模，作为"黑盒噪声"接受
- [6]-[7]: 这才是 GEM 的主场，我们的独家创新
```

### 2.3 各预测终点的精确定义（端到端用词的澄清）

"端到端"一词在不同语境下指不同的事，必须精确区分：

```
ADA 链路上所有可能的预测终点：

终点 A: 预测 [4] MHC-II 呈递
    = "这段肽能不能被 MHC-II 呈递？"
    代表工具：NetMHCIIpan
    数据量：IEDB 几万到几十万条
    难度：★（已经做得很好）

终点 A': MAPPs epitope load
    = "用质谱测，蛋白上被呈递多少段肽？"
    代表数据：MassIVE MSV000085375
    Barra 2020 已用此作训练标签
    难度：★★

终点 B: 预测 [5]-[6] T 细胞激活
    = "这段肽能不能激活 T 细胞？"
    代表工具：TLimmuno2、DeepImmuno、ImmunoStruct
    数据量：IEDB 几千条
    难度：★★（AUC 0.85+）

终点 B': 体外 T 细胞 readout
    = "PBMC 体外实验里 T 细胞被激活到什么程度？"
    代表指标：IL-2 分泌（Day 3）、增殖（Day 7）
    数据量：可从文献整理几十到上百条
    难度：★★

终点 C: 预测 [6]-[7] T 细胞分化方向
    = "这段肽激活的 T 细胞更倾向 Tfh 还是 Treg？"
    代表工具：【空白！项目目标位置】
    数据量：几乎没有公开配对数据
    难度：★★★

终点 D: 预测 [9] 临床 ADA 发生率
    = "上市后病人有多大比例产生 ADA？"
    代表工具：EpiVax ISPRI（仅限单抗）
    数据量：< 30 对治疗性多肽配对
    难度：★★★★★
```

### 2.4 项目的真实位置

```
项目要做什么（精确定义）：

【我们要做的端到端模型】
    输入：多肽序列 + HLA-II 类型
    输出：免疫原性风险评分
    
【对标的终点】
    True Question 想答 = 终点 C（Tfh vs Treg 倾向）
    实际训练标签 = 终点 B' + 终点 A'
    
    因为终点 C 没有直接公开数据，
    用 B'（体外 readout）作为代理标签训练，
    用 A'（MAPPs）作为辅助标签

【不做什么】
    终点 D（临床 ADA）：数据严重不足
    全虚拟 T 细胞：工程量过大
    非天然氨基酸：数据瓶颈无法克服
```

## 3-4. GEM 是不是必需？有别的机理融合思路吗？

### 3.1 用 GEM 的核心好处

```
GEM 提供的独特能力：

[1] 把"T 细胞代谢状态"转化为可量化的特征向量
    ├─ 通量分布（每条代谢反应的速率）
    ├─ 通路活性（糖酵解/FAO/TCA 等的总体强度）
    └─ 关键代谢物的生成/消耗速率
    
[2] 这些特征有机理性 / 因果意义
    └─ 不是黑盒，Reviewer 能看懂为什么这个特征预测了那个结果
    
[3] 同一多肽 + 不同代谢环境 = 不同 ADA 风险
    └─ 这种差异性只能通过显式建模代谢状态来捕捉
    
[4] 与已有 T 细胞 GEM 工具兼容
    └─ CD4T1670 (Han 2016) + Sen 2021 亚型 GEM 可直接接入
```

### 3.2 不用 GEM 也能做什么

```
替代方案：

方案 A: 纯序列预测 + 直接预测 ADA
    工具：NetMHCIIpan + EpiVax 路线
    问题：完全不考虑细胞侧，已经接近性能天花板
    
方案 B: 用 scRNA-seq embedding 代替 GEM
    工具：scGPT / Geneformer
    问题：黑盒，机理性弱
    arXiv 系统评估：未超过简单基线
    Alan 方法论不接受纯黑盒
    
方案 C: 用 ODE 建模 mTOR-HIF-1α 信号通路
    工具：自己实现
    问题：信号通路 ≠ 代谢；只覆盖几条关键开关
    需要大量动力学参数拟合
    
方案 D: 用多尺度全集成（Wertheim 2021 思路）
    工具：GEM+ODE+布尔+ABM
    问题：工程量过大（5-10 人年）
    Alan 明确说"不要做大工程"
```

### 3.3 三层回答：GEM 必需性的真实评估

```
GEM 是必需的吗？分三层回答：

【第一层：如果目标是"做最好的 ADA 预测工具"】
    GEM 不是必需，甚至可能不是最优
    直接用 scRNA-seq + 表位预测可能更省事
    
【第二层：如果目标是"显式建模代谢状态对免疫原性的影响"】
    GEM 几乎是必需的
    这是把代谢状态转化为可解释特征的最成熟工具
    
【第三层：从 Alan / Eric 方法论看】
    要求"机理引擎 + AI 驱动"
    机理引擎不一定是 GEM，但 GEM 是目前最现成的代谢机理工具
```

【判断】GEM 是"当前最自然的选择"，**不是"逻辑上唯一选择"**。论文里应该明确说明**为什么选 GEM**，并对比若不用 GEM 会有什么损失。这是诚实科学态度。

### 3.4 GEM + ODE 混合的最优解（远期）

```
最优技术路径：
    
第一阶段（项目当前）：纯 GEM
    └─ 用 FBA 稳态分析
       现成模型 + 现成工具链（COBRApy）
       
第二阶段（论文需要更深时）：GEM + 简化 ODE
    ├─ GEM 处理代谢状态（稳态层面）
    ├─ ODE 处理 mTOR-HIF-1α 等代谢开关动态
    └─ 类似 Wertheim 2021 但精简版
    
第三阶段（远期愿景）：完整虚拟 T 细胞
    └─ 不在本项目范围
```

### 启发维度 B：因果链证据（项目理论基础）

```
┌─────────────────────────────────────────────────────────────┐
│ Shi LZ et al. 2011 J Exp Med ✅                            │
│ PMID: 21708926                                              │
│ + Dang EV et al. 2011 Cell（独立第二篇）                   │

│                                                              │
│ 对项目的意义：                                               │
│   项目核心假设"代谢状态可以预测 T 细胞分化方向"的实验基础    │
│   论文 Introduction 必引                                     │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ Mayberry CL et al. 2022 Front Immunol ✅【关键综述】       │
│ PMID: 35493515                                              │
├─────────────────────────────────────────────────────────────┤
│ 它是什么：Tfh 代谢的最系统综述                               │
│                                                              │
│ 重大发现（颠覆了项目早期假设）：                             │
│   - 成熟 GC-Tfh：低糖酵解 + 高 FAO + 高 OXPHOS              │
│   - Bcl-6 主动转录抑制糖酵解基因（GLUT1/HK2/PKM）           │
│   - 真实因果方向：低糖酵解（AMPK 激活）→ Bcl-6 去抑制 → Tfh │
│   - 高糖酵解反而抑制 Bcl-6 和 Tfh                           │
│                                                              │
│ 推论（重要！）：                                             │
│   Tfh 和 Treg 在主代谢轴上反而相似（都低糖酵解 + 高 FAO）   │
│   Tfh 和 Th17 反而代谢特征几乎相反                          │
│                                                              │
│ 对项目的意义：                                               │

### 8.3 训练标签策略

```
训练标签优先级（已查证）：

【优先级 1】IL-2 分泌（Day 3 PBMC assay）
    来源：Arata 2023 + Joubert 2016 + Walsh 2024 等
    数据规模：可整理 50-100 条 (peptide, IL-2 %响应) 配对
    优势：最快速、最强相关性
    
【优先级 2】PBMC T 细胞增殖（Day 7 CDI）
    来源：Walsh 2024 (45 mAb) + Ito 2019 (218 供体)
    数据规模：可整理 30-50 条
    
【优先级 3】FluoroSpot（IFN-γ / IL-5）
    来源：Schultz 2017 + Barra 2020 FluoroSpot 数据
    数据规模：可整理 20-30 条

【优先级 4】MAPPs epitope load
    来源：MassIVE MSV000085375 + Barra 2020 数据
    数据规模：可整理 10-20 个蛋白
    注意：Barra 2020 已用此方式训练，我们做组合用法

【明确不用】
    ✗ DC-T 重刺激（Walsh 2024 警告 false positive 增强）
    ✗ 临床 ADA 率（数据 < 30 对，仅作外部验证）

## 9. 风险与决策点

### 9.1 已知的硬性风险

```
风险 1：数据稀缺性【最高优先级】
    问题：体外 readout 数据需手工整理，规模可能 < 100 条
          MAPPs 数据规模有限（仅几个蛋白有完整数据）
    
    缓解策略：
        ├─ 系统整理多篇论文（Walsh 2024 / Arata 2023 / 
        │  Joubert 2016 / Schultz 2017 / Ito 2019）
        ├─ 用 MAPPs MassIVE 数据集补充
        └─ 复旦合作可能提供新数据（待确认）

风险 2：Tfh 因果链不完全清楚
    问题：糖酵解 → Bcl-6 → Tfh 的分子机制 2026 年仍不完全清晰
          用 Th17 代谢作参照可能引入系统偏差
    
    缓解策略：
        ├─ 用 Bcl-6 调控网络 + FAO 路径作为关键特征
        ├─ 论文明确声明这是"模型假设"
        └─ 在 Discussion 中诚实讨论局限

风险 3：scRNA-seq 数据的 FAO 基因检出率
    问题：10x 3' 文库对低表达 FAO 基因（CPT1a/HADHA）
          dropout 率高
          这刚好是 Tfh 代谢的核心通路
    
    缓解策略：
        ├─ 首选：bulk RNA-seq + scRNA-seq 互补
        ├─ 备选：Smart-seq2 数据（细胞数少但深度高）
        └─ 退路：复旦合作做专项 Smart-seq2

风险 4：IVIVC 噪声天花板
    问题：体外 assay 敏感性 53-71%
          意味着即使完美预测体外 readout
          预测临床 ADA 的敏感性天花板也在此范围
    
    缓解策略：
        ├─ 论文 limitations 必须明确声明
        ├─ 评估指标用敏感性/特异性而非 r 值
        └─ 强调"代谢特征对体外 readout 的增益"作为主要贡献

风险 5：竞争者追赶
    问题：Oxford POEM、DTU Nielsen 组、EpiVax 都可能扩展
    
    缓解策略：
        ├─ 时间窗口 1-2 年内完成 MVP
        ├─ 开源 + 同行评议是差异化护城河
        └─ Tfh GEM 是难以快速复现的独家资产
```
