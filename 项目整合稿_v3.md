## [0] 项目背景简述

项目最初的目标是为多肽药物开发一套 ADMET 性质预测平台，受到组里 PhD 陈帅华（Eric, CS 背景）和陈藤（Alan, 医学背景）所做的"GEM + 虚拟细胞建模"工作启发。Eric 提出方法论："机理引擎 + AI 驱动方式"，即把基因组尺度代谢模型（GEM）作为内在的机理引擎，AI 是驱动这个引擎的方式。这一思路与纯数据驱动的 AI 模型有本质区别，也与纯机理建模不同。

项目方向经过了几个阶段的演变和收敛：

```
最初目标：多肽 ADMET 平台
    ↓
[排除 1] 药物性肝损伤 (DILI)
    原因：多肽不走 CYP450 通路，被蛋白酶水解为天然氨基酸
         小分子的 DILI 机理链对多肽不成立；多肽 DILI 数据 < 30 条
    ↓
[排除 2] 溶血、肠道渗透性、代谢稳定性
    原因：GEM 增益不成立
         - 溶血是膜物理作用，GEM 无因果贡献
         - PepT1 只转运二/三肽，对治疗性多肽无效
         - 多肽降解主要在细胞外，GEM 建模细胞内代谢，空间错位
    ↓
[关键发现] 多肽体内代谢的真实图景
    通过系统调研发现：多肽主要在血浆和肾脏被蛋白酶水解
                    肝脏不是主要代谢器官（Alan 一句话点破"虚拟肝细胞是伪命题"）
    ↓
[转向] 免疫原性（ADA, anti-drug antibody）
    Yuming 在组会上提出：做免疫细胞的虚拟细胞，预测多肽免疫原性
    Alan 和 Eric 均认可，评价"从 0 到 1 的创新性"
    Alan 进一步指令：构造虚拟 T 细胞，预测多肽-T 细胞结合
    ↓
当前方向：虚拟 T 细胞 + 多肽免疫原性预测
```

项目转向免疫原性方向有几个关键合理性：

- **免疫原性是 ADMET 的 T**（Toxicity），不偏离 ADMET 目标
- **ADA 同时影响 ADME 的每个字母**：加速清除（E）、降低有效暴露（D）、缩短半衰期（M）、影响 SC 吸收（A）
- **免疫原性是多肽特有挑战**：小分子几乎没有 ADA 问题
- **GEM 增益真实且因果性强**：T 细胞活化的代谢重编程直接决定 Tfh/Treg 分化方向 → 直接影响 ADA 产生
- **空白领域**：未发现将 T 细胞代谢通量作为免疫原性预测变量的成熟方法

本调研报告的目的是把项目当前的科学定位、方法选择、领域全景、差异化空间、MVP 设计、风险与决策点系统化梳理一遍，作为下一步实际工作的起点。

## 第一部分：科学定位

### [1] 为什么是 T 细胞 —— 特性、前景与挑战

#### T 细胞在 ADA 形成中的核心地位

ADA（抗药抗体）的产生不是 B 细胞独立完成的。绝大多数针对治疗性多肽 / 蛋白的 ADA 都是"T 细胞依赖性"的（Td-ADA, T-cell dependent ADA），意思是 B 细胞必须先获得 CD4+ T 细胞的辅助，才能产生针对外源蛋白的高亲和力抗体。**T 细胞是 ADA 产生链路上的瓶颈节点**：没有 T 细胞激活，就不会有 ADA。

具体来说，ADA 形成的完整链路是这样的：

```
[1] 多肽药物进入体内
        ↓
[2] APC（抗原呈递细胞，主要是树突细胞 DC、巨噬细胞）通过内吞摄取多肽
        ↓
[3] APC 在内体/溶酶体中用 cathepsin 把多肽切割成 13-25 aa 片段
        ↓
[4] 切割后的片段加载到 MHC-II 分子上，运送到 APC 表面展示
        ↓
[5] CD4+ T 细胞用 TCR 检查 APC 展示的肽-MHC-II 复合物
        ↓
[6] 若 TCR 识别 + 共刺激信号到位，T 细胞活化、增殖
        ↓
[7] 活化的 T 细胞分化成不同亚型（Tfh / Treg / Th1 / Th2 / Th17）
        ↓
[8a] 若分化为 Tfh（滤泡辅助 T 细胞）→ 跑到淋巴结生发中心帮 B 细胞
        ↓
[9a] B 细胞活化、亲和力成熟、类别转换 → 产生 ADA

或者：

[8b] 若分化为 Treg → 抑制免疫反应 → 不产生 ADA
```

#### CD4+ T 细胞的几个亚型角色

CD4+ T 细胞活化后会分化成不同"功能角色"，每种角色对 ADA 的影响完全不同：

| 亚型 | 全称 | 主要功能 | 对 ADA 的影响 |
|------|------|----------|--------------|
| **Tfh** | T follicular helper | 在淋巴结滤泡内帮助 B 细胞 | **直接驱动 ADA** |
| **Treg** | regulatory T cell | 抑制免疫反应，维持耐受 | **降低 ADA** |
| Th1 | T helper 1 | 应对细胞内病原（病毒） | 中等影响 |
| Th2 | T helper 2 | 应对寄生虫、过敏 | 中等影响 |
| Th17 | T helper 17 | 应对真菌、炎症反应 | 间接影响 |

**关键判别对**：**Tfh vs Treg**。同一个多肽，如果在体内更倾向激发 Tfh，就高 ADA 风险；如果更倾向激发 Treg，就低 ADA 风险。这对决策是项目预测的科学终点。

#### 学界的研究现状

T 细胞免疫原性预测在学界已经形成了几个层级的工具体系：

```
工具层级（从浅到深）：

[MHC-II 呈递预测层]
    NetMHCIIpan / MixMHC2pred / IEDB tools
    任务：肽 + HLA-II → 结合亲和力
    成熟度：高（AUC > 0.85）
    
[T 细胞激活预测层]
    TLimmuno2 / DeepImmuno / ImmunoStruct
    任务：肽 + HLA → T 细胞激活概率
    成熟度：中高（TLimmuno2 独立测试集 AUC ≈ 0.74；
            注：常被引用的 0.87 是训练集交叉验证结果，
            已核实为高估，不代表真实泛化性能）
    
[T 细胞分化方向预测层]  ← 项目目标
    几乎空白
    
[临床 ADA 预测层]
    EpiVax ISPRI（仅针对单抗，商业不公开）
    成熟度：商业声明但无同行评议性能数据
```

学界正在快速向"虚拟免疫细胞"概念推进，但仍属画饼阶段：

- **Oxford Virtual Immune Cells**（Kennedy Trust 博士项目）：提出"机制 + 单细胞 + AI"的混合方法，目标预测组织中 T 细胞应答。**仍在招生阶段，无公开工具**
- **CZI Virtual Immune System**：Chan Zuckerberg Initiative 2025 年 11 月宣布的大型项目，未来 10 年投资 100 亿美元。**目前已发布的相关模型都是通用单细胞工具，无 T 细胞特化模块**

#### 前景与挑战

**前景**：

1. **真正空白**：把 T 细胞代谢状态显式作为免疫原性预测特征，目前无任何已发表工作
2. **机理足够成熟**：HIF-1α-糖酵解检查点（Shi 2011 J Exp Med, PMID 21708926）+ AMPK-FAO-Treg 轴等代谢-命运因果链已有强实验证据
3. **国际趋势对齐**：Oxford POEM 和 CZI VIS 都瞄准"虚拟免疫细胞"，项目方向与国际前沿一致
4. **工具基础已具备**：CD4T1670（Han 2016）、Sen 2021 亚型 GEM 提供了代谢骨架

**挑战**：

1. **数据稀缺**：治疗性多肽-ADA 配对数据极少（公开 < 30 对），临床 ADA 数据碎片化
2. **Tfh 因果链不完全清晰**：糖酵解 → Bcl-6 → Tfh 这条链的分子机制截至 2026 年仍不完全清楚（详见后文）
3. **IVIVC（体内-体外相关性）噪声大**：体外 T 细胞应答 → 临床 ADA 之间有显著噪声（B 细胞阶段引入约 30-50% 变异）
4. **Tfh-Treg 代谢相似性**：在主代谢轴（糖酵解-OXPHOS）上 Tfh 反而接近 Treg，而非接近 Th17。这对模型特征工程提出更高要求（后文详述）

### [2] T 细胞的什么过程 —— ADA 链路 + GEM 主战场定位

#### ADA 链路上各环节与现有工具对照

```
完整 ADA 形成链路：

[1] 多肽进入        ──────  无需建模
        ↓
[2] APC 吞噬         ──────  无现成预测工具，但已知机理
        ↓
[3] APC 切割          ──→  Cathepsin 数据库 / 体外实验
                            (NetMHCIIpan 隐式考虑加工)
        ↓
[4] MHC-II 呈递       ──→  NetMHCIIpan-4.3（计算预测）
                            MAPPs 实验（质谱实测）
                            Barra 2020 (NNAlign_MAC)
        ↓
[5] T 细胞识别        ──→  TLimmuno2 / IEDB Immunogenicity
                            DeepImmuno (但仅 MHC-I)
        ↓
[6] T 细胞活化   ─────┐
                       ├──→ 几乎空白（项目主战场）
[7] T 细胞分化   ─────┘
                       
        ↓
[8] Tfh 辅助 B        ──────  机理研究，无计算工具
        ↓
[9] B 细胞产 ADA      ──→  EpiVax ISPRI（仅商业，仅单抗）
                            临床 ADA 数据极少
```

#### 各终点的精确定义

为避免歧义，我们把链路上的"终点"系统编号：

| 终点编号 | 位置 | 定义 | 数据可获得性 |
|---------|------|------|------------|
| **终点 A** | [4] | MHC-II 呈递（计算预测） | 高（IEDB 数万对） |
| **终点 A'** | [4] | MHC-II 呈递（MAPPs 实验实测） | 中（MassIVE 等数据集） |
| **终点 B** | [5]-[6] | T 细胞激活（计算预测） | 中（IEDB 数千对） |
| **终点 B'** | [5]-[6] | T 细胞激活（PBMC 体外 readout 实测） | 中低（文献整理几十到上百条） |
| **终点 C** | [7] | T 细胞分化方向（Tfh vs Treg） | 几乎无直接公开数据 |
| **终点 D** | [9] | 临床 ADA 发生率 | 极低（< 30 对治疗性多肽） |

**项目的科学终点是 C**（T 细胞分化方向）。但由于 C 没有直接数据，项目实际用 **B'（体外 readout）** 和 **A'（MAPPs）** 作为训练标签来间接逼近 C。

#### GEM 在 ADA 链路上的工作位置

GEM（基因组尺度代谢模型）作为一种代谢建模工具，它能描述的是细胞内部各代谢反应的通量分布。在 ADA 链路上，GEM 真正能发挥作用的只有 **[6]-[7] 这一段**：

```
GEM 能不能建模？各环节对照：

[1] 多肽进入                  ❌ 不涉及代谢
[2] APC 吞噬                  ❌ 膜运输，非代谢
[3] APC 切割                  ⚠️ 蛋白水解，GEM 粒度粗
[4] MHC-II 呈递               ❌ 蛋白质装配，非代谢
[5] T 细胞识别                ❌ 信号转导，非代谢
[6] T 细胞活化   ←────────── ✅ 代谢重编程（6-72h）
                              GEM 的主战场 #1
                              
[7] T 细胞分化   ←────────── ✅✅ 代谢-命运因果链
                              GEM 的主战场 #2
                              （HIF1α/AMPK 轴有铁实验证据）

[8] Tfh 辅助 B                ❌ 跨细胞互作
[9] B 细胞产 ADA              ❌ 复杂下游过程
```

#### 项目主战场的扩展含义

把"主战场是 [6]-[7]"这件事讲清楚之后，还有几个推论值得注意：

**推论 1：项目的科学故事必须以"T 细胞代谢决定分化方向"为核心**

这意味着论文中对项目的描述应该是：
- 不是"用 AI 预测 ADA"（这只是终点的近似）
- 而是"通过建模 T 细胞代谢状态来预测 CD4+ T 细胞应答的方向性"

**推论 2：[3]-[5] 段的工具我们直接用，不重新做**

NetMHCIIpan-4.3、TLimmuno2 等已有工具会作为我们流水线的"序列侧零件"，我们的工作不是替代它们，而是在它们之上增加"细胞侧"特征。

**推论 3：[8]-[9] 段的不可预测性必须诚实声明**

项目的预测停在 [6]-[7]（中游终点），与最终临床 ADA（[9]）之间还有 B 细胞激活、生发中心反应、同种型转换等步骤的噪声。论文中必须明确：项目输出的是"T 细胞应答倾向"，与临床 ADA 有相关性但不是直接预测。

**推论 4：项目主战场的代谢窗口是 6-72 小时**

这是 T 细胞活化后从代谢重编程到亚型分化的时间窗口。这一窗口决定了：
- 数据需求：覆盖这一时间段的 scRNA-seq 或 bulk RNA-seq 数据
- GEM 建模：可以用"分阶段稳态"近似，不必做完整动态 FBA
- 验证方式：Seahorse 数据测 OCR/ECAR 比值，可对照 GEM 预测

## 第二部分：方法选择

### [3] 怎么融入机理建模 —— 从机理建模总览到 GEM

#### 为什么需要"机理 + AI"融合

项目方法论的根基由 Eric 提出："机理引擎 + AI 驱动方式"。这一表述区别于两种极端：

```
三种建模哲学的对比：

[A] 纯数据驱动（黑盒 AI）
    输入特征 → 大量数据训练 → 模型预测
    优点：实现简单，性能可能很高
    缺点：不可解释，外推差，不能告诉你"为什么"

[B] 纯机理模型（白盒）
    用 ODE / GEM / ABM 等数学工具刻画生物过程
    优点：可解释，机理清楚
    缺点：参数难拟合，预测精度受限于机理理解程度

[C] 机理 + AI 融合（项目选择的路）
    机理模型负责刻画"已知机理"
    AI 负责学习"机理之上的复杂规律"
    优点：可解释 + 高性能 + 外推能力
    缺点：架构复杂，需要两套方法都熟悉
```

落到项目场景，"纯数据驱动"的代表是 EpiVax ISPRI（用大量 mAb-ADA 数据训练黑盒模型）。它的局限是：方法和数据都不公开，性能宣称（如"6 倍相关性提升"）无法独立验证，无法告诉你"为什么这个多肽 ADA 风险高"。**项目选择"机理 + AI"是有意的差异化**。

#### 现有机理建模方法全景

把可选的机理建模方法系统盘点一遍：

| 方法 | 核心思想 | 时间尺度 | 在 T 细胞场景的工作 | 我们能用吗 |
|------|---------|---------|-------------------|----------|
| **ODE（常微分方程）** | 描述变量随时间的连续变化 | 秒-小时 | mTOR-HIF-1α-c-Myc 信号通路 ODE 概念框架 | 可用作 GEM 输入约束的辅助 |
| **布尔网络** | 描述基因调控开关关系 | 离散 | Th 分化 GRN 模型（T-bet/GATA3/RORγt/Foxp3） | 可作概念参考 |
| **GEM（基因组尺度代谢模型）** | 用 FBA 求解代谢反应网络的稳态通量 | 稳态近似 | CD4T1670 / Sen 2021 亚型 GEM | **项目核心工具** |
| **ABM（基于代理模型）** | 模拟细胞群体的个体行为 | 细胞-组织级 | CARCADE（CAR-T 设计） | 不适合本项目 |
| **多尺度混合** | GEM + ODE + 布尔 + ABM 整合 | 跨尺度 | Wertheim 2021 多尺度 CD4+ T 模型 | 工程量过大，参考概念但不实做 |
| **scRNA-seq embedding + ML** | 用 scGPT/Geneformer 把转录组压缩成向量 | 单细胞快照 | scGPT 等通用工具 | 仅作"特征压缩"辅助，不作机理替代 |

#### 落到 T 细胞场景：为什么 GEM 是当前最自然的起点

```
逻辑链：

项目主战场 = T 细胞活化后的代谢重编程 + 分化方向决定
    ↓
"代谢重编程"是真正的代谢过程
    （糖酵解通量、FAO 通量、谷氨酰胺通量等都可量化）
    ↓
代谢通量在数学上最自然的建模工具就是 GEM
    （ODE 适合信号通路，GEM 适合代谢网络）
    ↓
T 细胞场景有现成可用的 GEM：CD4T1670 + Sen 2021
    ↓
代谢-命运因果链有实验证据：HIF1α → 糖酵解 → Th17 / FAO → Treg
    ↓
GEM 是当前最自然的起点
```

#### [3.1] GEM 是什么 —— 用通俗语言讲清楚

```
一个 GEM 就是一张"代谢反应清单"加一套"求解规则"。

类比：发动机蓝图 + 油门设置 → 各管路流速

具体来说：
    
[元素 1] 反应清单（数千条代谢反应）
    例如：葡萄糖 + ATP → 6-磷酸葡萄糖 + ADP
         （这是糖酵解第一步，由己糖激酶催化）
    
[元素 2] 化学计量矩阵 S
    把所有反应的"投入产出比"写成一个大矩阵
    
[元素 3] 边界约束
    设定外界条件：
    - 营养摄取上限（如葡萄糖摄取 ≤ 5 mmol/h）
    - 氧气供应
    - 关键代谢物的生产/消耗约束

[元素 4] 目标函数
    "细胞要最大化什么"
    例如：最大化 biomass 合成（增殖）
         或最大化 ATP 生产
         
[元素 5] FBA（通量平衡分析）求解
    在稳态假设下（代谢物浓度不变）
    求解每条反应的通量（速率）

输出：
    每条代谢反应的通量值（数千维向量）
    可以解读为这个细胞在当前环境下的"代谢指纹"
```

#### [3.2] T 细胞 GEM 的现状

```
T 细胞 GEM 的家谱：

Human-GEM（通用人代谢模板）
    ├─ ~13,000 反应，覆盖全身代谢通路
    └─ 项目的"基础模板"
        ↓
        加上 CD4+ T 细胞转录组约束
        ↓
        
CD4T1670 (Han 2016, PMID 26646474)
    ├─ 1670 个代谢反应（naive CD4+ T）
    ├─ 覆盖：糖代谢、TCA、脂肪酸代谢、谷氨酰胺代谢
    └─ 用途：FBA 模拟基因敲除、营养扰动
        ↓
        加上各亚型分化条件的转录组约束
        ↓
        
Sen 2021 亚型 GEM (Cell Reports, PMID 34758307)
    ├─ Th1 / Th2 / Th17 / iTreg 四个亚型
    ├─ 整合了脂质组学验证（鞘糖脂通路）
    └─ 没有 Tfh ← 项目要补的空白
        ↓
        ?
        ↓
        
Tfh GEM ← 项目原创贡献
    ├─ 文献完全空白（PubMed + bioRxiv 检索确认）
    ├─ 基于扁桃体 GC-Tfh scRNA-seq 或 bulk + Seahorse
    └─ 工作量估计：5-8 人月（无湿实验）
```

#### [3.3] GEM 在项目里的具体使用方式（数据流详解）

这一节回答"GEM 怎么用进去"这个最技术的问题。完整数据流如下：

```
完整 GEM 数据流：

[阶段 1] 准备代谢骨架
    
    Human-GEM（通用模板）
        + CD4T1670（naive T 细胞约束）
        + Sen 2021 方法学
        ↓
    构建几种代表性 T 细胞状态的 GEM：
        ├─ naive CD4+
        ├─ Th1 / Th2 / Th17
        ├─ Treg / iTreg
        └─ Tfh ← 项目独家自建

[阶段 2] 从 scRNA-seq 到 context-specific GEM
    
    输入：人扁桃体 scRNA-seq（如 E-MTAB-13687）
          或 cTfh / GC-Tfh 排序的 bulk RNA-seq
        ↓
    数据预处理：QC + 聚类 + pseudo-bulk 聚合
        ↓
    用 tINIT 算法约束 Human-GEM 模板
        （tINIT 根据转录组表达水平
         保留高表达基因对应的代谢反应
         移除几乎不表达的反应）
        ↓
    输出：context-specific Tfh GEM

[阶段 3] FBA 求解得到通量
    
    用 COBRApy（Python 工具包）
    输入：
        - context-specific GEM
        - 营养条件（葡萄糖、谷氨酰胺、脂肪酸浓度）
        - 信号通路输入（mTOR / HIF-1α 活性作为软约束）
        - 目标函数（biomass / ATP / 自定义）
        ↓
    求解 FBA 优化问题
        ↓
    输出：每条反应的通量值（约 1000-2000 维向量）

[阶段 4] 通量 → 特征向量
    
    原始通量（数千维）
        ↓
    按 KEGG / Reactome 通路汇总
        （把数千条反应聚合成 ~100 个通路的活性指标）
        ↓
    主成分分析 / 自编码器降维
        ↓
    "代谢状态 embedding"（紧凑向量，~50 维）

[阶段 5] 接入预测模型
    
    肽序列侧特征
        ├─ NetMHCIIpan-4.3 输出（结合分数）
        ├─ TLimmuno2 输出（T 细胞激活概率）
        └─ ESM/ProtBERT embedding
                ↓
                与代谢状态 embedding 拼接
                ↓
        融合预测网络（MLP / cross-attention）
                ↓
        输出：免疫原性风险评分
```

#### 关键设计决策

数据流之外，有几个核心设计决策必须想清楚：

**决策 A：模型推理时只有肽序列，怎么知道 T 细胞会进入什么代谢状态？**

```
问题：训练时我们有 scRNA-seq 数据可以约束 GEM，
      但模型部署后用户只输入肽序列，
      没有 scRNA-seq，怎么计算代谢特征？

解决方案：预先训练几种"代表性代谢状态"GEM
    
    模型推理流程：
        输入：多肽序列
            ↓
        分别在几种预先准备好的代谢状态下做预测：
            ├─ 状态 1：Tfh-promoting 环境（低糖酵解 / 高 FAO）
            ├─ 状态 2：Treg-promoting 环境
            ├─ 状态 3：Th17-promoting 环境
            └─ 状态 4：naive 静息状态
            ↓
        每种状态都给一个评分
            ↓
        最终输出 = 各状态评分的加权 / 最大值
                  或学习一个聚合函数
    
    输出含义：
        "这个多肽在不同免疫环境下的风险谱"
        既反映"激发强度"也反映"激发方向"
```

**决策 B：静态 GEM 还是动态 GEM？**

```
项目第一版用静态（稳态 FBA）：
    优点：方法成熟，CD4T1670/Sen 2021 都是稳态
    缺点：不直接捕捉 0-72h 的时间动态

如何近似时间动态：
    用"分阶段稳态"代替动态 FBA
        ├─ 早期（0-6h）：OXPHOS 为主稳态
        ├─ 中期（6-24h）：糖酵解+OXPHOS 共存稳态
        └─ 晚期（24-72h）：亚型分化稳态
    
    每个阶段用对应的转录组数据约束 GEM
    生成 3 个时间快照的代谢特征
    
第二阶段扩展（如果论文需要更深）：
    引入简化 ODE 描述 mTOR-HIF1α 动力学
    与 GEM 耦合（动态 FBA / dFBA）
```

**决策 C：Tfh GEM 怎么构建？**

```
三层策略：

[优先] 用公开扁桃体 scRNA-seq + tINIT
    数据：E-MTAB-13687（HCA 扁桃体细胞图谱）
    工作量：5-8 人月
    风险：FAO 基因（CPT1a / HADHA 等）在 10x 3' 文库 dropout 严重
    
[退路 1] bulk RNA-seq + Seahorse 数据
    数据：GSE157801（GC-Tfh bulk RNA-seq）
    优点：bulk 数据代谢基因覆盖更好
    缺点：失去单细胞异质性
    
[退路 2] 复旦免疫系合作生成 Smart-seq2 数据
    工作量：1-1.5 年（伦理 + 样本 + 测序）
    成本：15-25 万 RMB
    优点：数据为项目量身定制
    缺点：周期最长
    
决策：先走优先路径，下载 E-MTAB-13687 验证 FAO 基因覆盖
      若不达标，启动 bulk RNA-seq 互补策略
```

#### [3.4] GEM 的局限和补救

诚实地讲清楚 GEM 的局限，避免在论文中过度承诺：

```
GEM 的三大局限：

[局限 1] 稳态假设
    GEM 求解的是稳态通量，不直接刻画时间动态
    补救：分阶段稳态 + 后续可加 ODE 耦合

[局限 2] 信号通路不在范围内
    TCR 信号、mTOR 通路、HIF-1α 调控等
    都不是代谢反应，不在 GEM 反应清单里
    补救：把信号通路活性作为 GEM 输入的"软约束"
         比如"mTOR 激活时，糖酵解上限放大 X 倍"
         这部分需要拟合关键参数

[局限 3] 不刻画细胞内空间分布
    GEM 把细胞当成"均质大锅"，不区分细胞器
    补救：用区室化 GEM（部分支持）或不补救
         本项目第一版不需要这种精细度
```

#### 训练标签优先级

```
训练标签分三层（按可获得性 + 有效性排序）：

[第一层] IL-2 分泌 / T 细胞增殖 (PBMC assay)
    数据源：从论文整理（Walsh 2024, Arata 2023, 
            Joubert 2016, Ito 2019 等）
    规模：几十到上百条
    质量：高（直接反映 T 细胞激活强度）
    优先级：★★★★★

[第二层] MAPPs epitope load
    数据源：MassIVE MSV000085375 (Rooney 2020)
            + 其他 MAPPs 论文补充材料
    规模：取决于覆盖蛋白数
    质量：中（呈递 vs 激活有差距）
    优先级：★★★★

[第三层] 临床 ADA 率（仅作外部验证）
    数据源：FDA labels + 文献报告
    规模：< 30 对治疗性多肽
    质量：低（数据极少，噪声大）
    优先级：★（不作训练，仅作验证）
```

### [9] 风险与决策点

#### 项目硬性风险

```
[风险 1] Tfh 代谢-命运因果链不完全（科学风险）
────────────────────────────────────────────
描述：糖酵解 → Bcl-6 → Tfh 分子机制截至 2026 年仍不
     完全清楚。代谢调控可能是必要但非充分条件。

影响：
    - GEM 增益可能比预期低
    - 模型预测的"Tfh-like 倾向"可能噪声大

缓解：
    - 用多通路联合特征（不只糖酵解）
    - 论文 Discussion 诚实声明
    - 准备退路：即使 Tfh 预测精度不高
      项目仍有"Tfh GEM 资产" + "IVIVC 量化" 两大独立贡献


[风险 2] Tfh-Treg 代谢主轴相似（特征工程风险）
────────────────────────────────────────────
描述：Mayberry 2022 综述揭示 Tfh 和 Treg 在主代谢轴
     上反而相似（都是低糖酵解、高 FAO）

影响：
    - 简单代谢特征可能分不开 Tfh 和 Treg
    - 模型需要更精细的特征

缓解：
    - 用 FAO 子通路 + CDP-乙醇胺 + 脂质代谢联合
    - 引入 Bcl-6 vs Foxp3 调控网络相关代谢节点
    - 用消融实验定位关键判别通路


[风险 3] 数据稀缺（数据风险）
────────────────────────────────────────────
描述：体外 T readout 数据需手工整理，规模 < 几百
     MAPPs 数据集有限
     临床 ADA 数据极少

影响：
    - 模型容易过拟合
    - 泛化性难验证

缓解：
    - 用迁移学习从 MHC-II 结合预训练（参考 TLimmuno2）
    - 数据增强（合成阴性样本）
    - 严格的交叉验证 + 留出外部验证集


[风险 4] FAO 基因 scRNA-seq dropout（构建 Tfh GEM 的风险）
────────────────────────────────────────────
描述：10x 3' Chromium 文库对 CPT1a / HADHA / ACADL 等
     低表达 FAO 基因检出率不足
     而 FAO 是 Tfh 的核心代谢特征

影响：
    - Tfh GEM 的核心通路可能约束不准

缓解：
    - 优先用 bulk RNA-seq（GSE157801 等）作 FAO 通路补充
    - 用 pseudo-bulk 聚合多细胞提升基因检出
    - Smart-seq2 数据如可获取则优先用
    - 必要时启动复旦合作生成专项数据


[风险 5] IVIVC 噪声天花板（性能风险）
────────────────────────────────────────────
描述：体外 readout → 临床 ADA 的相关性有限
     Walsh 2024 显示敏感性仅 53%
     模型即使完美预测体外，预测临床 ADA 仍有天花板

影响：
    - 论文中需要诚实声明这一限制
    - 不能宣称项目能精确预测临床 ADA

缓解：
    - 论文卖点围绕"体外 readout 预测"
    - 把临床 ADA 作为外部验证而非训练目标
    - 量化 IVIVC 噪声本身作为副产品发表


[风险 6] 竞争对手提前发表（时间风险）
────────────────────────────────────────────
描述：DTU (Barra 2020 团队)、刘雪松组（TLimmuno2）
     都有可能扩展到与项目重合的领域

注意（经核实更新）：
    Oxford POEM 做的是 MHC-I / CD8+ 通路，
    和本项目（MHC-II / CD4+ / Tfh）不是正面竞争，
    已从竞争对手名单中移除

影响：
    - 项目原创性可能被部分稀释（来自DTU和刘雪松组）

缓解：
    - 早期发布 preprint 占住时间戳
    - 不必等完整论文，先公开 Tfh GEM 资产
    - 三维交叉空间（治疗性多肽+MHC-II+GEM代谢）难被一家通吃
```

### [12] 免疫学机理待确认问题清单

> 以下问题直接影响建模方案，需要免疫学专业判断。
> 来源：调研过程中识别出的建模假设中的不确定点。
> 用途：与复旦免疫系合作讨论时的问题清单。

---

**问题1：短肽能不能绕过APC加工直接呈递**

```
背景：
    完整ADA链路要求多肽被cathepsin切割成片段再装上MHC-II
    但治疗性多肽本身是10-40aa，MHC-II结合槽容纳13-25aa
    理论上短肽可能不需要切割就能直接装上MHC-II

这对建模影响极大：
    若必须切割 → 滑窗方法合理（把长肽切成15aa片段分别预测）
    若短肽可直接呈递 → 可以不滑窗，直接处理全长肽，精度更高

需要判断：
    "对于10-40aa的治疗性多肽，APC的cathepsin切割
    在体内是必须发生的吗？有没有实验证据说明
    直接呈递 vs 切割后呈递各自的比例？"

证据强度：△ 仅推测，需专业确认
```

**问题2：体外Tfh分化的可靠判定标准**

```
背景：
    我们需要体外实验数据作为ML分类器的训练标签
    "体外Tfh"的判定标准在文献里不统一

标志物选择直接决定训练标签定义，
用了不可靠的标志物，整个ML训练的标签都是错的

需要判断：
    "PBMC体外实验中，判断T细胞向Tfh分化
    通常用什么标志物组合最可靠？
    CXCR5+ PD-1high 够吗？还是必须加BCL-6蛋白表达？
    IL-21分泌能否单独作为readout？
    从刺激到看到稳定分化一般需要多少天？"

证据强度：⚠️ 有文献但不统一，需专业确认
```

**问题3：Tfh和Treg在代谢上真的能区分吗**

```
背景：
    这是整个项目GEM模块的核心假设
    Mayberry 2022 综述揭示：
    成熟GC-Tfh：低糖酵解 + 高FAO + 高OXPHOS
    Treg：低糖酵解 + 高FAO + 高OXPHOS（TGF-β通路）
    两者在主代谢轴上高度相似

    如果代谢上区分不开Tfh和Treg
    GEM通量特征就没有区分力
    → 项目核心假设崩塌

需要判断：
    "Tfh和Treg在代谢层面是否有可以被GEM建模的可区分差异？
    谷氨酰胺代谢、脂肪酸链长偏好、特定氨基酸消耗速率
    这些维度有没有已知的可测差异？
    有没有同时比较Tfh和Treg代谢组学的实验文献？"

证据强度：⚠️ 主轴相似已确认，细分差异待确认
```

**问题4：HLA多态性对治疗性多肽免疫原性的影响**

```
背景：
    不同HLA型的人对同一多肽免疫应答差异很大
    我们的模型需要选择覆盖哪些HLA-DR等位基因
    默认的NetMHCIIpan参数针对欧美人群

需要判断：
    "面向中国人群，哪些HLA-DR等位基因对
    GLP-1类或类似治疗性多肽的呈递贡献最大？
    体外实验设计中，应选多少个HLA型的供体
    才能覆盖足够的中国人群代表性？"

证据强度：△ 仅一般原则，具体数据待确认
```

**问题5：体外T细胞激活数据对临床ADA的预测力**

```
背景：
    Walsh 2024 (PMID 38863708) 显示
    PBMC体外assay对单抗：特异性100%，敏感性53%
    这个数字决定了用体外readout作为训练标签的合理性

需要判断：
    "对于治疗性多肽，体外T细胞增殖阳性
    和临床ADA之间的相关性如何？
    假阴性（体外阴性但临床有ADA）的主要原因是什么？
    这个IVIVC对多肽是否比单抗更好或更差？"

证据强度：△ 单抗数据已知，多肽数据几乎为零
```
